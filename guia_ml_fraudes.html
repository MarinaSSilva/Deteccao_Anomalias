MÓDULO 01 · PREPARAÇÃO DOS DADOS
Feature Engineering
Transformar variáveis brutas em representações que os modelos conseguem aprender melhor.
Por que transformar o Amount? conceito

O valor das transações tem uma distribuição muito assimétrica: a maioria é pequena (R$ 10, R$ 50), mas algumas são altíssimas (R$ 25.000). Essa diferença de escala prejudica modelos lineares.

log1p resolve isso: aplica log(1 + x) e comprime os valores grandes, deixando a distribuição mais simétrica. O "+1" evita problemas com transações de R$ 0,00.

// visualização: efeito do log1p na distribuição
import numpy as np
df["Amount_log"] = np.log1p(df["Amount"])
# log1p(0)   → 0   (sem erros com zeros)
# log1p(10)  → 2.4
# log1p(1000) → 6.9  (comprime valores extremos)
StandardScaler — colocando tudo na mesma escala sklearn

Algoritmos como Regressão Logística são sensíveis à escala. Se Amount varia de 0 a 25.000 e V1 varia de -5 a 5, o modelo vai se deixar dominar pelo Amount — não porque ele é mais importante, mas porque o número é maior.

StandardScaler transforma cada variável para ter média = 0 e desvio padrão = 1.

z = (x − μ) / σ
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
# fit_transform: aprende a média/std NO TREINO e transforma
df["Amount_Scaled"] = scaler.fit_transform(df[["Amount"]])

# ⚠️ No teste: apenas .transform() — nunca .fit_transform()
x_test["Amount_Scaled"] = scaler.transform(x_test[["Amount"]])

Undersampling — reduzir a classe majoritária estratégia 1

Sorteia aleatoriamente a mesma quantidade de transações legítimas que existem fraudes, criando um dataset 50/50.

# Separar as classes

normais = df[df["Class"] == 0].sample(

Vantagem: simples e rápido. Desvantagem: perde 99,65% dos dados legítimos — pode descartar padrões importantes.

SMOTE — criar fraudes sintéticas estratégia 2

Synthetic Minority Oversampling Technique: ao invés de copiar fraudes existentes, o SMOTE cria novas fraudes interpolando entre vizinhos mais próximos no espaço de features.

// visualização: SMOTE criando pontos sintéticos
from imblearn.over_sampling import SMOTE

smote = SMOTE(random_state=42)
X_res, y_res = smote.fit_resample(x_train, y_train)

# Antes: 199.020 legítimas, 344 fraudes
# Depois: 199.020 legítimas, 199.020 fraudes sintéticas

Como funciona: para cada fraude real, o SMOTE escolhe k vizinhos mais próximos (outras fraudes) e cria um novo ponto aleatório entre eles. Não é cópia — é interpolação.

MÓDULO 03 · ALGORITMOS
Modelos de Machine Learning
Três algoritmos com abordagens distintas para o mesmo problema.
Regressão Logística — o baseline linear

Calcula a probabilidade de uma transação ser fraude através de uma combinação linear das features, transformada pela função sigmoide (resultado entre 0 e 1).

P(fraude) = σ(β₀ + β₁V1 + β₂V2 + ... + βₙAmount) = 1 / (1 + e^−z)
from sklearn.linear_model import LogisticRegression

model = LogisticRegression(max_iter=1000)
# max_iter=1000: mais iterações para convergir num dataset grande
model.fit(x_train, y_train)
y_pred = model.predict(x_test)

Quando usar: primeiro modelo a testar sempre. Rápido, interpretável, bom baseline. Limitado a relações lineares.

Random Forest — ensemble de árvores independentes ensemble

Treina N árvores de decisão em paralelo, cada uma em uma amostra aleatória dos dados e features. A predição final é a votação da maioria.

// random forest: votação entre árvores
from sklearn.ensemble import RandomForestClassifier

rf = RandomForestClassifier(
    n_estimators=50,          # 50 árvores no ensemble
    max_depth=10,             # profundidade máxima (evita overfitting)
    class_weight="balanced",  # penaliza + erros na classe fraude
    n_jobs=-1                 # usa todos os núcleos da CPU
)

class_weight="balanced" faz o modelo prestar mais atenção às fraudes: cada erro numa fraude pesa proporcionalmente mais que um erro numa transação legítima.

XGBoost — gradient boosting sequencial boosting

Diferente do Random Forest, o XGBoost treina árvores em sequência: cada nova árvore foca nos erros da anterior. O resultado final é a soma ponderada de todas as árvores.

Métricas de Avaliação
Acurácia sozinha mente. Entenda o que realmente importa em detecção de fraudes.
A matriz de confusão — base de tudo fundamento

FNFalse NegativePreviu legítima, era fraude🚨 Fraude passou!

As métricas — por que cada uma importa fórmulas

Recall (Sensibilidade) — mais crítico em fraudes

Recall = TP / (TP + FN)

De todas as fraudes reais, quantas detectamos? FN = fraude que passou = prejuízo.

Precision

Precision = TP / (TP + FP)

Das que dissemos ser fraude, quantas realmente eram? FP = cliente bloqueado.

F1-Score

F1 = 2 × (P × R) / (P + R)

Média harmônica entre Precision e Recall. Útil quando há desbalanceamento.

Acurácia — enganosa aqui!

Acc = (TP + TN) / Total

Um modelo idiota que nunca detecta fraude tem 99,83% de acurácia.

GridSearchCV
Testar todas as combinações de hiperparâmetros para encontrar o modelo ótimo.
Hiperparâmetros vs Parâmetros conceito

Escolher hiperparâmetros na mão é subjetivo e lento. GridSearchCV automatiza isso: testa todas as combinações possíveis e diz qual é a melhor.

Como funciona o GridSearch algoritmo
// grid de combinações testadas
from sklearn.model_selection import GridSearchCV

param_grid = {
    "max_depth": [3, 5],          # 2 opções
    "n_estimators": [50, 100]    # 2 opções → 4 combinações
}

grid = GridSearchCV(
    XGBClassifier(eval_metric="logloss"),
    param_grid,
    scoring="recall",  # ← CRUCIAL: otimizar pelo que importa
    cv=3,             # 3-fold cross-validation
    n_jobs=-1        # paralelo
)
grid.fit(x_train, y_train)
print(grid.best_params_)  # melhor combinação
print(grid.best_score_)   # melhor recall obtido
Cross-Validation — por que não basta um split? conceito

Com cv=3, o conjunto de treino é dividido em 3 partes iguais. O modelo é treinado 3 vezes: cada vez usando 2 partes para treino e 1 para validação. O score final é a média das 3 rodadas.

// cross-validation com cv=3

Isso garante que o score não depende de qual parte específica do dado foi usada como validação — resultado mais confiável e generalizado.

Com scoring="recall", o GridSearch escolhe os parâmetros que maximizam o Recall — que é o que importa para detecção de fraude.

MÓDULO 06 · BOAS PRÁTICAS
Pipeline sklearn
Encadear pré-processamento e modelo num único objeto — e evitar o data leakage.
O que é Data Leakage? armadilha crítica

Data leakage acontece quando informações do conjunto de teste "vazam" para o treino, inflando artificialmente as métricas. O modelo parece ótimo no teste, mas falha no mundo real.

O caso clássico: aplicar o StandardScaler no dataset inteiro antes do split. A média e o desvio padrão do scaler foram calculados incluindo os dados de teste — o modelo "viu" o teste indiretamente.

❌ SEM Pipeline — data leakage:

scaler.fit_transform(X_all) → split → treinar modelo
O scaler conhece os dados de teste. Resultado: métricas otimistas demais.
✅ COM Pipeline — sem leakage:

split → Pipeline.fit(X_train) → Pipeline.predict(X_test)
O scaler só aprende com o treino. Resultado: avaliação honesta.
Pipeline em ação código

O Pipeline encadeia etapas na ordem certa. No .fit(), ele executa todas em sequência com o treino. No .predict(), aplica as transformações aprendidas ao novo dado.

X_treino
→
StandardScaler
.fit_transform()
→
LogisticRegression
.fit()
→
Modelo treinado
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression

pipeline = Pipeline([
    ("scaler", StandardScaler()),          # etapa 1
    ("model",  LogisticRegression(max_iter=1000))  # etapa 2
])

pipeline.fit(x_train, y_train)
# → scaler.fit_transform(x_train)
# → model.fit(x_train_scaled, y_train)

y_pred = pipeline.predict(x_test)
# → scaler.transform(x_test)  ← .transform() apenas, não .fit()!
# → model.predict(x_test_scaled)
Pipeline + GridSearchCV — a combinação ideal avançado

Você pode passar um Pipeline diretamente ao GridSearchCV. Use o prefixo nome_etapa__parametro para referenciar parâmetros das etapas.

param_grid = {
    "model__max_depth": [3, 5],
    "model__n_estimators": [50, 100]
}    # ↑ prefixo "model__" referencia a etapa do pipeline

pipeline_xgb = Pipeline([
    ("scaler", StandardScaler()),
    ("model",  XGBClassifier(eval_metric="logloss"))
])

grid = GridSearchCV(pipeline_xgb, param_grid, scoring="recall", cv=3)
grid.fit(x_train, y_train)
# O scaler nunca vê o fold de validação. Zero leakage.
